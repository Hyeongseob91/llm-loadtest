# Clarify 질문 템플릿

## 카테고리별 질문 예시

### 1. 인증/보안 관련

| 모호한 표현 | 명확화 질문 | 선택지 예시 |
|------------|------------|------------|
| "로그인 만들어줘" | 인증 방식은? | Username/Password, OAuth, Magic Link, SSO |
| "보안 강화해줘" | 어떤 보안 요소? | HTTPS, CORS, Rate Limiting, Input Validation |
| "권한 관리해줘" | 권한 모델은? | Role-based (RBAC), Permission-based, Attribute-based |

### 2. UI/UX 관련

| 모호한 표현 | 명확화 질문 | 선택지 예시 |
|------------|------------|------------|
| "버튼 추가해줘" | 버튼 위치는? | 헤더, 푸터, 사이드바, 콘텐츠 영역 |
| "예쁘게 만들어줘" | 디자인 스타일은? | 미니멀, 모던, 글래스모피즘, 기존 테마 유지 |
| "반응형으로" | 지원 기기는? | 모바일 우선, 데스크탑 우선, 태블릿 포함 |

### 3. 데이터/상태 관련

| 모호한 표현 | 명확화 질문 | 선택지 예시 |
|------------|------------|------------|
| "데이터 저장해줘" | 저장 위치는? | 로컬 스토리지, 세션, 서버 DB, 쿠키 |
| "캐싱해줘" | 캐시 전략은? | 메모리, Redis, CDN, Service Worker |
| "동기화해줘" | 동기화 방식은? | 실시간 (WebSocket), 폴링, 수동 새로고침 |

### 4. 에러/예외 처리

| 모호한 표현 | 명확화 질문 | 선택지 예시 |
|------------|------------|------------|
| "에러 처리해줘" | 에러 표시 방식은? | Toast, Modal, Inline, Console only |
| "실패하면" | 재시도 정책은? | 자동 재시도 (3회), 사용자에게 물어봄, 조용히 실패 |
| "로깅해줘" | 로그 레벨은? | Debug, Info, Warn, Error only |

### 5. API/통신 관련

| 모호한 표현 | 명확화 질문 | 선택지 예시 |
|------------|------------|------------|
| "API 만들어줘" | API 스타일은? | REST, GraphQL, gRPC, WebSocket |
| "실시간으로" | 실시간 방식은? | WebSocket, SSE, Long Polling |
| "연동해줘" | 연동 패턴은? | 직접 호출, Proxy 경유, Message Queue |

### 6. 테스트/품질 관련

| 모호한 표현 | 명확화 질문 | 선택지 예시 |
|------------|------------|------------|
| "테스트 작성해줘" | 테스트 종류는? | Unit, Integration, E2E, 모두 |
| "검증해줘" | 검증 방식은? | 스키마 검증, 타입 체크, 비즈니스 룰 |
| "품질 올려줘" | 품질 지표는? | 커버리지, 복잡도, 중복 제거, 성능 |

## 라운드별 질문 전략

### Round 1: 핵심 방향 결정
가장 큰 분기점이 되는 질문부터 시작합니다.

```
예: "인증 기능"
→ "OAuth vs 자체 인증" (이 선택이 모든 후속 결정에 영향)
```

### Round 2: 세부 구현 결정
Round 1 답변을 기반으로 구체화합니다.

```
예: "OAuth 선택"
→ "어떤 OAuth 제공자?" (Google, GitHub, Apple 등)
→ "추가 정보 수집?" (프로필, 이메일만, 없음)
```

### Round 3: 엣지 케이스 및 마무리
예외 상황과 사용자 경험을 확정합니다.

```
예: "Google OAuth + GitHub"
→ "두 계정 동일 이메일이면?" (연결, 별도 유지, 거부)
→ "로그아웃 동작은?" (현재 세션만, 모든 기기)
```

```json
      "options": [
        {
          "label": "OAuth (Recommended)",
          "description": "Google, GitHub 등 외부 인증. 구현 간단, 보안 위임."
        },
        {
          "label": "Username/Password",

    }
  ]

```

## 저장 형식 템플릿

명확화된 요구사항을 저장할 때 사용하는 형식:

```markdown
# [기능명] 명세서

## 생성일
YYYY-MM-DD HH:MM

## 원본 요청
> "[사용자 원본 요청]"

## 명확화된 요구사항

### 기술 선택
- **[항목]**: [선택값]

### 동작 명세
- **[시나리오]**: [동작]

### 예외 처리
- **[케이스]**: [처리 방법]

## 참고
- [추가 컨텍스트]
```
